なぜ作ったか

多くの推薦は「モデルを作った」で終わる。プラットフォームの仕事はその先 —
行動ログとコンテンツを特徴量にし、索引で検索・推薦を出し、品質とコストを測ること。

A-B シミュレーション(KPI = hit-rate@10)

A = content   B = collaborative   (n=80 users)

KPI_A 0.7000 → KPI_B 0.9000   lift +0.2000 (+28.6%)

95% bootstrap CI (B−A): [+0.0875, +0.3250]   判定: B wins

区間が勝敗を裏付ける — 点推定が正でも CI が 0 を跨げば「判定不能」と正直に出す。

正直な負の結果

このワークロードでは hybrid は collab に勝たない。鋭い信号を融合が薄めるため。
ハーネスがそれを可視化し、本番は collab を選ぶ — 盲目的な融合はしない。

この判断は ADR-0009 に記録。技術選定はすべて ADR 0001〜0009 で説明可能。

コストと性能

Qdrant のローカルモードで 384 次元検索を p50 約 5.9ms → 0.45ms(約 13 倍)。
埋め込みは民生 CPU で毎秒およそ 310 件 — GPU 不要。
